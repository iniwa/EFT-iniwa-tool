## 1. 広告枠コンポーネントの雛形

CLS (Cumulative Layout Shift) 対策のため、広告タグを入れる前に **レイアウトを先に確定** させておく。

- 配置候補: フッター直上 / `ResultList.vue` 末尾
- `min-height` を確保した空コンテナを先に配置するだけで OK
- ロード後の差し替えトランジションは不要 (CLS 防止が主目的)

## 2. AdSense 申請 → CMP 導入 → ads.txt 配置 → 広告タグ埋込

着手順:

1. **AdSense アカウント作成 → サイト審査申請**
2. **審査通過後** に CMP (Cookie 同意管理) を導入
   - Funding Choices など Google 認定 CMP を採用予定
   - 同意取得まで広告タグをロードしない実装が必須 (GDPR / 改正電気通信事業法)
3. **`public/ads.txt` を配置**
   - 過去に削除した経緯あり (commit `bb736f1`)。審査通過後に再配置する
4. **広告タグの埋込**
   - 1. で用意した枠コンポーネントに差し替える
   - Privacy ページの「将来 AdSense を導入する可能性がある」記述は導入時に「導入済み」へ書き換える

## 3. 導入時に忘れない補足

- Privacy ページの AdSense 関連記述は v3.1.2 投入済み → **本文差し替えだけで対応可能**な状態
- About ページに「Battlestate Games とは無関係」を明記済み → **商標トラブル懸念は低減済み**
- 広告投入後は Umami の `Tab Switch` 集計を継続観察し、PV / 滞在時間が落ちていないか確認する
